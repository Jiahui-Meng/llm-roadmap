# Day 84 — Diagrams and Visuals

## 1. 为什么到作品集阶段必须补图

前面你已经有很多文字和代码了。

但当项目越来越复杂时，只靠文字说明会有一个问题：
- 理解成本高
- 关键结构不够直观
- 面试时不好快速讲清楚

所以 Day 84 的重点不是“美化仓库”，而是：

> **用图把复杂系统压缩成一眼可懂的结构表达。**

---

## 2. 为什么图比长文更适合展示系统

对于像这些项目：
- Transformer block
- RAG pipeline
- Research Copilot workflow
- Health Agent hybrid retrieval

如果只用纯文字说明，读者需要自己在脑子里组装结构。

而图的作用是：
- 直接暴露关键模块
- 展示数据流 / 控制流
- 突出系统边界
- 帮助快速讲解

所以 diagrams and visuals 的本质是：

> **提升系统表达效率。**

---

## 3. 哪些图最值得做

不是所有图都同样重要。

最值得优先补的通常是：
- 核心架构图
- 数据流图
- workflow 图
- 关键对比图

比如：
- mini Transformer block 图
- RAG v2 pipeline 图
- Research Copilot workflow graph
- Health Agent dense + graph + timeline retrieval 图

这些图都能显著提升项目可读性。

---

## 4. 为什么图不仅服务读者，也服务你自己

当你尝试把系统画出来时，你会很容易发现：
- 模块边界还不清楚
- 数据流有断点
- 某些概念在 README 里解释过于含混
- 系统叙事不够统一

所以画图不仅是对外展示，也是一种对内澄清。

这和写 README 很像：

> **图是在逼你做结构化思考。**

---

## 5. 为什么 Day 84 对面试尤其有帮助

在面试里，很多项目讲不清，不是因为项目不够好，而是：
- 结构太复杂
- 没有一张图能快速压缩表达

如果你有一张清楚的架构图，你可以很快讲：
- 输入是什么
- 中间模块是什么
- 输出是什么
- 亮点在哪里

所以 Day 84 实际上也是面试准备的一部分。

---

## 6. 今天最该记住的 5 句话

1. **图不是美化，而是高密度表达系统结构的工具。**
2. **复杂项目只靠长文通常不够直观。**
3. **最值得做的是架构图、数据流图和 workflow 图。**
4. **画图会反向帮助你理清系统边界。**
5. **Day 84 对项目展示和面试表达都非常重要。**
