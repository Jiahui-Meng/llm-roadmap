# Day 41 — Query Rewrite\n\n## Why rewrite queries\nUser queries are often short, vague, or conversational. Retrieval systems work better when the query is rewritten into a retrieval-friendly form.\n\n## What rewrite helps with\n- expand missing keywords\n- clarify intent\n- normalize terminology\n- create multiple search variants\n\n## Risk\nRewrite can drift from the original intent, so it should be evaluated carefully.\n\n## One-line summary\nQuery rewrite improves retrieval by translating user language into search language, but it must preserve intent.\n
