# Day 85 — Resume Bullets

## 1. 为什么项目做完之后还要写 resume bullets

很多人做完项目后，会停在：
- 代码写了
- README 写了
- GitHub 也整理了

但如果这些内容不能被压缩成简历语言，那么它们在求职中的价值会大幅缩水。

所以 Day 85 的重点不是“写几句话”，而是：

> **把项目成果转译成招聘市场能理解的价值表达。**

---

## 2. resume bullet 在做什么

resume bullet 不是项目摘要，也不是 README 复制。

它真正做的是：
- 说明你做了什么
- 说明你用到了什么能力
- 说明结果或价值是什么

也就是说，它是在把：
- 技术过程

转成：
- 招聘语境下的能力信号

所以 Day 85 本质上是一种翻译工作。

---

## 3. 为什么 bullet 不能只写“我学了什么”

简历里最没力量的一类句子就是：
- 学习了 Transformer
- 了解了 RAG
- 熟悉了 LoRA

这些句子的问题是：
- 太抽象
- 没有动作
- 没有产出
- 没有结果

更强的 bullet 应该强调：
- 设计了什么
- 实现了什么
- 评估了什么
- 优化了什么

所以 Day 85 的关键意识是：

> **简历 bullet 讲的是行动与结果，不是“看过什么”。**

---

## 4. 为什么中英双语 bullet 都值得准备

如果你未来会同时面向：
- 中文岗位
- 英文简历投递
- 国际化公司

那中英双语 bullet 会非常有价值。

因为它不仅方便你直接投递，也能逼你用更清晰的语言重新理解项目。

特别是在 LLM / AI 领域，双语表达还能帮助你：
- 统一技术术语
- 锻炼项目讲述能力
- 为面试中的双语切换做准备

---

## 5. 为什么 Day 85 也是项目抽象能力训练

当你试图把一个项目压缩成一条 bullet 时，你必须回答：
- 这个项目最核心的贡献是什么？
- 它展示的能力究竟是什么？
- 哪一部分最值得被写进简历？

这实际上是一种非常高强度的抽象训练。

所以 Day 85 不只是“求职准备”，也是：

> **对项目价值的再次提炼。**

---

## 6. 今天最该记住的 5 句话

1. **resume bullets 是把项目成果翻译成招聘市场语言。**
2. **bullet 应强调行动、技术和结果，而不是“学了什么”。**
3. **强 bullet 要有动词、有系统对象、有价值指向。**
4. **中英双语 bullet 能提升求职灵活性和项目表达力。**
5. **Day 85 也是一次项目价值提炼训练。**

---

## 7. 今日任务

### 必做
1. 为每个核心项目写 2–3 条 bullet
2. 每条 bullet 都包含动作 + 技术 + 结果/价值
3. 准备一版中文和一版英文
4. 写一句：你的项目中最值得上简历的亮点是什么

### 你要能回答的问题
1. 为什么项目做完还要写 resume bullets？
2. bullet 和 README 的区别是什么？
3. 为什么 bullet 不能只写“学习了什么”？
4. 中英双语 bullet 为什么值得准备？
5. Day 85 为什么也是抽象能力训练？

---

## 8. 一句话总结

> Resume Bullets 的本质，是把项目中的技术动作、系统设计和工程成果压缩成招聘语境下高密度、可感知、可比较的能力信号，从而真正把项目价值转化成求职价值。
