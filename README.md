# llm-roadmap

A structured roadmap for going from Transformer foundations to production-grade LLM engineering.

## Target Profile

This repo is designed for an engineer who:
- already has solid software / ML engineering background
- wants to move into LLM / GenAI / Applied AI roles
- needs a practical, project-oriented roadmap
- wants daily tasks, reading lists, implementation work, and portfolio outputs

## Main Goals

By the end of this roadmap, you should have:
- solid Transformer and LLM foundations
- working understanding of RAG, vLLM, LoRA/QLoRA, SGLang, and Agent workflows
- engineering-grade project assets
- portfolio-ready repos and reports
- interview-ready notes and explanations

## Structure

- `docs/roadmap-overview.md` — overall 12-week plan
- `docs/daily-plan.md` — full day-by-day execution plan
- `docs/progress-tracker.md` — checkbox tracker for all days
- `docs/daily-template.md` — copyable daily note template
- `docs/weekly-review-template.md` — weekly review template
- `docs/how-to-use-this-repo.md` — execution guide
- `docs/resources.md` — papers, blogs, docs, and tools
- `docs/links.md` — direct clickable reading links by topic
- `docs/projects.md` — flagship project plans and dataset ideas
- `docs/future-topics.md` — important follow-up topics not yet added to the main plan
- `notes/` — daily and weekly notes
- `code/` — foundational implementations and experiments
- `experiments/` — retrieval, tokenization, serving, and tuning experiments
- `bridges/` — notes that connect theory to engineering systems
- `projects/` — project scaffolds for portfolio-grade work
- `reports/` — benchmarks, evaluations, and retrospectives

## Suggested Progression

1. Transformer foundations
2. Modern LLM architecture and inference basics
3. RAG and retrieval evaluation
4. vLLM serving and benchmarking
5. LoRA / QLoRA fine-tuning
6. SGLang and structured generation
7. Agent workflows and productization
8. Portfolio packaging and resume-ready outputs

## Working Rules

- Every study day should produce at least one artifact:
  - a note
  - code
  - experiment result
  - benchmark
  - or a diagram
- Prefer fewer, deeper materials over endless browsing.
- Always connect theory to practical LLM engineering.
- Keep commits small and meaningful.
- Use `scripts/start_day.sh` to create a fresh daily note from template.
- Use `docs/progress-tracker.md` as the execution dashboard.

## Recommended Daily Time Budget

For a 5–6 hour workday:
- 1.5h theory / reading
- 2h coding / implementation
- 1.5h project / engineering output
- 0.5–1h recap / documentation

## Final Expected Portfolio

- Transformer foundations repo assets
- RAG prototype + advanced RAG evaluation
- vLLM serving benchmark
- LoRA/QLoRA fine-tuning mini-project
- structured generation / SGLang lab
- Agent workflow prototype
- Health Agent 2.0 and/or Open Research Copilot
